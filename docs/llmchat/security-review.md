## Per-probe results

For reopened QA, `STATIC PASS / LIVE PENDING` means the unit/static defense-layer checks pass, but the attack surface is not yet a senior-security PASS because the clean-compose/live payload path is still missing. Under `LLMCHAT_SECURITY_REQUIRE_LIVE=1`, those probes now exit 77 and the orchestrator exits non-zero.

### Probe 01 — Delegation token scope

- **Payload:** request a chat-assistant child delegation with extra `allowed_actions=[cordum_update_policy_bundle]` and `allowed_topics=[cfg.*]`.
- **Expected defense layer:** `core/auth/delegation.TokenService` rejects widened child scopes (`ErrScopeExceeded`) and excessive chain depth (`ErrChainTooDeep`); `core/llmchat.DelegationClient` requests only the canonical chat-assistant MCP-tool allowlist and an empty topic set.
- **Actual:** STATIC PASS / LIVE PENDING. Static assertions verified `expectedAllowedTools()` is used, `allowed_topics` is not widened, policy-bundle mutation is excluded, TTL defaults to 15 minutes, and monotonicity/chain-depth tests pass.
- **Tests:** `go test ./core/auth/delegation ...`; `go test ./core/llmchat -run TestDelegationClient_...`.
- **Notes:** Current local branch expresses delegation actions as MCP tool names, not abstract `read/submit_job` verbs. The probe also accepts the predecessor narrow abstract-action contract so the security gate is stable across the in-flight delegation-scope patch. Mutating tools beyond submit are still approval-gated by Probe 03.

### Probe 02 — Agent identity scope

- **Payload:** crafted JSON-RPC tool call for a non-allowlisted tool such as `cordum_unlist_tool_xyz`.
- **Expected defense layer:** MCP `FilterForIdentity` removes non-visible tools before policy evaluation; caller sees JSON-RPC `-32601` method-not-found rather than an approval/policy denial.
- **Actual:** STATIC PASS / LIVE PENDING. MCP identity-filter tests and llm-chat bootstrap scope tests pass.
- **Tests:** `go test ./core/mcp -run TestFilterForIdentity_...|TestUnknownMethod`; `go test ./core/llmchat -run TestBootstrapper|TestExpectedAllowedTools|TestExistingChatAssistant`.

### Probe 03 — Preapproved mutation exploit

- **Payload:** from the chat path, attempt `cordum_update_policy_bundle`, `cordum_approve_job`, `cordum_reject_job`, `cordum_cancel_job`, and `cordum_trigger_workflow`; compare with `cordum_submit_job` baseline.
- **Expected defense layer:** only `cordum_submit_job` is preapproved; other mutations produce `approval_required` and audited `approval_status=required`.
- **Actual:** STATIC PASS / LIVE PENDING. Bootstrap pins `PreapprovedMutatingTools=[cordum_submit_job]`; gateway approval-gate and MCP audit tests pass.
- **Tests:** llm-chat bootstrap tests, gateway approval-gate tests, MCP invocation/audit tests, approval resume tests.

### Probe 04 — Prompt injection

- **Payloads:** role-mimicry tokens, XML-looking tool-call text, instructions to ignore redaction, Unicode homoglyph/RTL tool names, and staged multi-turn secret-dump roleplay. Placeholder hosts only (`attacker.example`, `evil.test`).
- **Expected defense layer:** system prompt remains authoritative, tool execution still goes through MCP/approval gates, and `DefaultRedactor` scrubs secret-shaped tool results before LLM context insertion.
- **Actual:** STATIC PASS / LIVE PENDING. The harness asserts guardrail prompt content, no unapproved state-changing path, and redaction of a controlled fake-secret fixture before context reinsertion.
- **Tests:** `go test ./core/llmchat -run TestTurn_RedactsSecretsInToolResults|...`; `go test ./core/mcp -run TestDefaultRedactor_...`; transient redactor fixture check.

### Probe 05 — Parser config pinning / DoS hardening

- **Payload:** malicious operator tries `--set qwenInference.toolCallParser=qwen3_coder` or swaps the parser in Compose/Helm.
- **Expected defense layer:** Compose/release and Helm render `qwen3_xml` only; disallowed parser aliases do not appear in deploy templates; `--disable-log-requests` remains present.
- **Actual:** PASS after remediation in this task. The first RED run caught Helm parser value-drivenness; the template now hardcodes `qwen3_xml`, and the lint renders with a malicious override to prove it stays pinned.
- **Tests:** `llmchat_probe_05_parser_pinning.sh`, `vllm_config_lint_test.sh`, `vllm_config_lint.sh`, `vllm_helm_lint.sh`.
- **Finding disposition:** fixed before handoff; no open P0/P1.

### Probe 06 — Loopback binding / zero external exposure

- **Payload:** change Compose port to wildcard/bare host binding or Helm Service to an externally reachable type.
- **Expected defense layer:** Compose publishes `127.0.0.1:8000:8000`; Helm Service renders `ClusterIP`; CI fails closed for wildcard/bare/external exposure.
- **Actual:** STATIC PASS / LIVE PENDING. Static checks, docker compose config, and Helm render checks pass. Live curl evidence is opt-in on a dedicated clean stack.
- **Tests:** `llmchat_probe_06_loopback_binding.sh`, `vllm_config_lint.sh`, `vllm_helm_lint.sh`.

### Probe 07 — Session hijack

- **Payload:** steal a `session_id` and attempt resume with a different principal/tenant or spoofed headers.
- **Expected defense layer:** trusted-forwarder and session-binding checks reject untrusted/spoofed resumes; forged session IDs do not create replacements.
- **Actual:** STATIC PASS / LIVE PENDING. Trusted principal/tenant resume and spoofing regression tests pass.
- **Tests:** `go test ./core/llmchat -run TestChatHandlers_ResumeRequires...|TestChatHandlers_DefaultIdentity...|TestChatHandlers_ForgedSessionID...`; `go test ./cmd/cordum-llm-chat -run TestRequireTrustedForwarder|TestRuntimeConfig`.

### Probe 08 — Admin page authZ

- **Payload:** non-admin GET `/api/v1/chat/sessions`, then admin GET for baseline.
- **Expected defense layer:** `requirePermissionOrRole("chat.read_all")`/permission checker denies non-admin with 403; admin path returns paginated sessions.
- **Actual:** STATIC PASS / LIVE PENDING. Admin list/detail/cross-tenant tests and fail-closed missing-checker tests pass.
- **Tests:** `go test ./core/llmchat -run TestChatAdmin...`.

### Probe 09 — WS origin / CSRF

- **Payload:** browser upgrade from `Origin: https://attacker.example` with a stolen token.
- **Expected defense layer:** gateway CORS/origin allowlist rejects malicious browser origins before the direct llm-chat WS handler; direct service also requires trusted forwarding.
- **Actual:** STATIC PASS / LIVE PENDING. Gateway origin parsing/CORS tests pass. Internal llm-chat upgrader accepts proxied origins by design; the browser-facing defense is the gateway.
- **Tests:** `go test ./core/controlplane/gateway -run TestCorsMiddleware|TestAllowedOriginsFromEnv|TestRequestHostname`.

### Probe 10 — Rate limiting / backpressure

- **Payload:** 100 messages/sec to one chat session for 60 seconds.
- **Expected defense layer:** gateway rate-limit middleware emits 429 or WS error frames; llm-chat enforces message-size/backpressure/turn budgets without panic or goroutine leaks.
- **Actual:** STATIC PASS / LIVE PENDING. Static/unit-backed guard tests pass, but the required 100 msg/sec live flood has not been run on an owned clean stack.
- **Tests:** gateway rate-limit middleware tests; llm-chat oversized WS message, wall-clock, assistant-byte budget, and repeat-call detector tests.

### Probe 11 — Log redaction

- **Payload:** synthetic `sk-test-*`, JWT-looking, `Bearer ...`, and cloud-key-shaped strings in tool results/prompts.
- **Expected defense layer:** redactor removes secret values before LLM context; vLLM request logging is disabled; log grep finds zero secret-like values after load.
- **Actual:** STATIC PASS / LIVE PENDING. Redactor and config guard tests pass and `--disable-log-requests` is enforced in Compose/release/Helm, but the required one-hour live load plus log grep has not been run.
- **Tests:** llm-chat redactor/provider tests, MCP `DefaultRedactor` tests, gateway auth log tests, config grep checks.
- **Finding disposition:** log-request suppression added before handoff; no open P0/P1.

### Probe 12 — Entitlement bypass

- **Payload:** flip `LLMChatAssistant=false` and attempt `/api/v1/chat/*`; confirm dashboard hides button.
- **Expected defense layer:** internal llm-chat entitlement gate returns stable feature-unavailable response; gateway proxy requires entitlement before forwarding; dashboard health/entitlement gate hides the header button within 10 seconds.
- **Actual:** STATIC PASS / LIVE PENDING. Licensing defaults, llm-chat handler gates, gateway proxy gates, and dashboard health/button tests pass.
- **Tests:** `go test ./core/licensing`; `go test ./core/llmchat -run TestChatHandlers_LicenseGate...`; gateway proxy tests; dashboard hook/button tests.
